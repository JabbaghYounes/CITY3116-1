1

Broken Access Control

A01:2025

Users or attackers can access resources or perform actions they shouldn't be authorised to do.

Network Relevance

APIs, session tokens, role checks, and direct object references can be bypassed, leading to unauthorised access and lateral movement across networks.

➡️
2

Security Misconfiguration

A02:2025

Incorrect or insecure default settings, open services, excessive privileges, and inconsistent controls.

Network Relevance

Exposed services and open ports increase the attack surface; misconfigured TLS or firewall rules enable network reconnaissance and exploitation.

➡️
3

Software Supply Chain Failures

A03:2025 (NEW)

Vulnerabilities originating from third-party dependencies, build systems, CI/CD pipelines and distribution tools.

Network Relevance

Compromised libraries or CI/CD pipelines can introduce malicious code traversing the network, trusted repositories becoming vectors for remote exploitation.

➡️
4

Cryptographic Failures

A04:2025

Weak, missing, or obsolete cryptography, poor key management.

Network Relevance

Encryption failures allow eavesdropping, man-in-the-middle (MITM) attacks, replay attacks and network traffic tampering.

➡️
5

Injection

A05:2025

Input that alters queries or commands (SQL, OS, template injection).

Network Relevance

Injected payloads traverse TCP/IP stacks and can manipulate underlying networked services or databases; malformed packets can disrupt network protocols.

➡️
6

Insecure Design

A06:2025

Security defects introduced through flawed architecture and lack of threat modelling.

Network Relevance

Poor design choices lead to insecure network interactions, trust boundaries too broad, APIs without segmentation, or unmanaged endpoints exposed over networks.

➡️
7

Authentication Failures

A07:2025

Weak authentication mechanisms, poor session handling, credential issues.

Network Relevance

Broken authentication allows credential stuffing, session hijacking, unauthorised access and lateral network movement.

➡️
8

Software/Data Integrity Failures

A08:2025

Code, data, or artefacts lacking integrity checks, susceptible to tampering.

Network Relevance

Tampered updates or data can circulate across networks, propagating compromised behaviour or backdoors across services.

➡️
9

Security Logging & Alerting Failures

A09:2025

Lack of effective logging and alerting to detect and respond to attacks.

Network Relevance

Without network monitoring and alerting, lateral movement by attackers and anomaly patterns remain unseen, delaying detection and response.

➡️
10

Mishandling of Exceptional Conditions

A10:2025 (NEW)

Improper error and failure handling including failing open during abnormal conditions.

Network Relevance

Attackers can leverage edge cases or error paths to bypass checks, crash services (leading to DoS) or expose internal data, affecting network controls.
