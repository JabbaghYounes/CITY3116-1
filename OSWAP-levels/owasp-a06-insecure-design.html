#6 Insecure Design

A06:2025 - Security defects from flawed architecture and lack of threat modeling

Common Design Flaws

🔓 Improper Trust Boundaries

APIs and services communicate without proper authentication between internal systems

🚫 Missing Rate Limiting

No limits on password attempts, API calls, or resource consumption

🌐 Lack of Network Segmentation

All services on flat network with no isolation between zones

🔑 Hardcoded Secrets in Design

Architecture relies on shared secrets embedded in code

Network Relevance

Poor design choices lead to insecure network interactions, trust boundaries too broad, APIs without segmentation, or unmanaged endpoints exposed over networks. Without proper threat modeling, attackers can exploit architectural weaknesses to move laterally across the network and access sensitive systems that should be isolated.
