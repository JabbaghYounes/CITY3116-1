Assignment Overview
Cyber-Physical Systems (CPS) are increasingly targeted by sophisticated cyber
threats, necessitating advanced security mechanisms. This assignment requires
students to design and implement an Intrusion Detection System (IDS) or Intrusion
Prevention System (IPS) for CPS security, incorporating Artificial Intelligence (AI)
and Machine Learning (ML) methodologies. Additionally, students will conduct
forensic analysis of security incidents using data-driven techniques.
Assignment Objectives
● Design and implement an AI/ML-driven IPS/IDS for CPS security.
● Analyse network traffic and detect threats using AI/ML algorithms.
● Apply forensic methodologies to investigate and mitigate security incidents.
● Evaluate the efficacy of AI/ML models in detecting cyber threats in real-time.
Assignment Tasks
Task 1: AI/ML-Driven Intrusion Detection System (IDS) or Intrusion Prevention
System (IPS) (50%)
You are required to design an IDS or IPS tailored explicitly for the security of
Cyber-Physical Systems. The system should incorporate AI or ML techniques to
enhance its ability to detect and respond to security threats effectively. The
implementation should involve selecting and training an appropriate AI/ML model,
such as supervised learning, anomaly detection, or reinforcement learning, using a
relevant cybersecurity dataset. The system should be evaluated based on its ability
to detect intrusions accurately and compared against traditional rule-based security
approaches to demonstrate its effectiveness. The final implementation should
provide insights into the advantages and limitations of using AI/ML in CPS security,
highlighting the challenges associated with real-time threat detection.
Task 2: Forensic Investigation & Incident Response (50%)
You must conduct a forensic investigation following a simulated cyber-attack on the
designed CPS security system. They should define an attack scenario, such as a
Man-in-the-Middle attack, Denial-of-Service, or ransomware infection, and capture
relevant forensic data, including network traffic logs and system artefacts. The
analysis should employ open-source forensic tools such as Wireshark, Volatility, or
Autopsy to examine the nature of the attack, assess security breaches, and evaluate
mitigation strategies. Additionally, students should explore AI/ML-based techniques
for forensic analysis, such as malware classification, anomaly detection, or forensic
log analysis. The investigation should culminate in a structured incident report
detailing the nature of the attack, identified anomalies, steps taken for mitigation, and
recommendations for improving the security posture of Cyber-Physical Systems.

Deliverables
● Technical Report (3000 words) detailing the design, implementation, and
evaluation of the IDS/IPS, forensic investigation findings, and security
recommendations.
● Source code and trained ML models (uploaded to GitHub or included as an
appendix).
● Presentation (5-10 slides) summarising key insights and findings.
Assessment Criteria
1. AI/ML Model Design & Implementation (30%) Effective use of AI/ML in
IDS/IPS, model accuracy, and dataset relevance.
2. Forensic Analysis (30%) Depth of investigation, correct use of forensic tools,
and quality of incident report.
3. Evaluation & Security Recommendations (20%) Critical assessment of
AI-driven security methods, mitigation strategies, and compliance with
standards (ISO 27001, OWASP Top 10).
4. Technical Report & Documentation (10%) Clarity, structure, justification of
methods, and technical accuracy.
5. Presentation & Communication (10%) Clear and concise summary,
effectiveness of delivery, and ability to answer questions.
