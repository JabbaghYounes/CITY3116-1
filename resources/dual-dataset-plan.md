# Dual-Dataset Implementation Plan: NSL-KDD + CIC-IDS2017

## Goal

Add CIC-IDS2017 dataset support to the CPS-IDS system and train three model variants:

- **Model A**: Trained on NSL-KDD only (existing pipeline, unchanged)
- **Model B**: Trained on CIC-IDS2017 only
- **Model C**: Trained on both datasets combined

This allows the writeup to demonstrate that the IDS ensemble generalises across datasets and time periods (NSL-KDD reflects 1999-era attacks; CIC-IDS2017 reflects 2017-era attacks).

---

## CIC-IDS2017 Dataset Profile

**Source**: University of New Brunswick, Canadian Institute for Cybersecurity
**Paper**: Sharafaldin, Lashkari & Ghorbani, "Toward Generating a New Intrusion Detection Dataset and Intrusion Traffic Characterization", ICISSP 2018
**Download**: https://www.unb.ca/cic/datasets/ids-2017.html (also mirrored on Kaggle and IEEE DataPort)

### Format

- CSV files generated by CICFlowMeter (bidirectional flow analysis)
- One file per day, 8 files total covering Monday–Friday (some days split AM/PM):
  - `Monday-WorkingHours.pcap_ISCX.csv`
  - `Tuesday-WorkingHours.pcap_ISCX.csv`
  - `Wednesday-workingHours.pcap_ISCX.csv`
  - `Thursday-WorkingHours-Morning-WebAttacks.pcap_ISCX.csv`
  - `Thursday-WorkingHours-Afternoon-Infilteration.pcap_ISCX.csv`
  - `Friday-WorkingHours-Morning.pcap_ISCX.csv`
  - `Friday-WorkingHours-Afternoon-PortScan.pcap_ISCX.csv`
  - `Friday-WorkingHours-Afternoon-DDos.pcap_ISCX.csv`
- Header row present (unlike NSL-KDD which has no header)
- ~2,830,743 total rows across all files
- ~78-80 columns: all numeric features + 1 label column

### Known Data Quality Issues

| Issue | Detail | Mitigation |
|-------|--------|------------|
| Column name whitespace | Headers have leading/trailing spaces (e.g. " Label", " Flow Duration") | Trim all header names on load |
| NaN / Infinity values | `Flow Bytes/s` and `Flow Packets/s` contain NaN/Inf when flow duration is zero | Replace with 0.0 |
| Non-numeric columns | Flow ID, Source IP, Destination IP, Timestamp are strings | Drop these 4 columns, keep ~74 numeric features |
| Label encoding | Some labels contain en-dash (U+2013) not regular hyphen: "Web Attack – XSS" | Use prefix matching (`starts_with("Web Attack")`) |
| DoS GoldenEye | Present in some versions of the dataset but not all | Include in label mapping |

### Feature Set (~74 numeric features after dropping non-numeric columns)

**Flow-level metrics**: Flow Duration, Total Fwd/Bwd Packets, Total Length of Fwd/Bwd Packets

**Packet size statistics**: Fwd/Bwd Packet Length Max/Min/Mean/Std, Min/Max/Mean/Std Packet Length, Packet Length Variance, Average Packet Size

**Inter-arrival time (IAT)**: Flow IAT Mean/Std/Max/Min, Fwd IAT Total/Mean/Std/Max/Min, Bwd IAT Total/Mean/Std/Max/Min

**TCP flag counts**: FIN, SYN, RST, PSH, ACK, URG, CWE, ECE Flag Counts, Fwd/Bwd PSH/URG Flags

**Throughput**: Flow Bytes/s, Flow Packets/s, Fwd/Bwd Packets/s

**Other**: Down/Up Ratio, Avg Fwd/Bwd Segment Size, Fwd/Bwd Avg Bytes/Packets/Bulk Rate, Subflow Fwd/Bwd Packets/Bytes, Init_Win_bytes_forward/backward, act_data_pkt_fwd, min_seg_size_forward, Active/Idle Mean/Std/Max/Min, Fwd/Bwd Header Length

| CIC-IDS2017 Label | Mapped Class | Rationale |
|--------------------|-------------|-----------|
| BENIGN | Normal (0) | Normal traffic |
| DoS slowloris | DoS (1) | Slow-rate DoS |
| DoS Slowhttptest | DoS (1) | Slow-rate DoS |
| DoS Hulk | DoS (1) | Volumetric DoS |
| DoS GoldenEye | DoS (1) | HTTP DoS |

This mapping aligns CIC-IDS2017's 15 labels with NSL-KDD's 5-class scheme so that all three models produce comparable evaluation metrics.

### Comparison with NSL-KDD

| Aspect | NSL-KDD | CIC-IDS2017 |
|--------|---------|-------------|
| Year of traffic | 1999 (KDD Cup) | 2017 |
| Total rows | ~150,000 | ~2,830,000 |
| Features | 41 (3 categorical + 38 numeric) → ~122 after one-hot | ~78 (all numeric) → ~74 after dropping non-numeric |
| Attack types | 4 categories, ~39 specific labels | 14 specific labels |
| Class balance | Relatively balanced | Heavily imbalanced (benign-dominated) |
| Feature style | Connection-level with content-based features (hot, num_failed_logins, logged_in) | Flow-level with bidirectional packet/timing statistics |
| Official split | Yes (KDDTrain+.txt / KDDTest+.txt) | No — single corpus, must self-split |

## Combined Model: Feature Space Design

### Problem

NSL-KDD and CIC-IDS2017 have different feature sets with limited overlap. NSL-KDD has ~122 features after one-hot encoding; CIC-IDS2017 has ~74 numeric features. A combined model needs a unified feature space.

### Approach: Union with Zero-Padding

Create a combined feature vector by concatenating both feature spaces:

```
Combined vector = [NSL-KDD features (122) | CIC-IDS2017 features (74)] = 196 total

NSL-KDD sample:     [actual_nsl_features | 0, 0, ..., 0 (74 zeros)]
CIC-IDS2017 sample: [0, 0, ..., 0 (122 zeros) | actual_cic_features]
```

Feature names prefixed `nsl_` and `cic_` to avoid collisions.

### Why This Works

- **Random Forest**: Tree splits naturally learn which features are informative. Zero-variance features in one dataset half are ignored. The RF effectively learns separate decision paths for each dataset's feature space.
- **Isolation Forest**: Random feature selection means it samples from both feature spaces. Constant-value (zero) columns for one dataset half produce leaf nodes immediately, so anomaly scoring adapts to whichever features are populated.
- **Simple to implement**: No complex feature alignment or mapping. Honest about the disjoint nature of the feature spaces.
- **Interpretable**: In the writeup, it's clear what the combined model is learning — it sees both 1999-era connection features and 2017-era flow features.

### Alternative Considered: Common Feature Mapping

Map both datasets to a shared ~30 feature set (duration, byte counts, packet counts, TCP flags, throughput). Rejected because:

- Only ~10 features truly overlap (duration, src/dst bytes, some flag counts)
- Discards most of each dataset's discriminative features
- Complex to implement and validate correctness of mappings
- Worse model performance due to information loss

## Sampling Strategy

CIC-IDS2017 has ~2.8M rows. The existing `RFModel` stores all training data in `train_x: Vec<Vec<f64>>` for JSON persistence (smartcore models aren't serializable). Memory and disk budget:

| Model | Training rows | Features | Memory (f64) | JSON disk |
|-------|--------------|----------|-------------|-----------|
| A (NSL-KDD) | ~100k | 122 | ~97 MB | ~200 MB |
| B (CIC-IDS2017) | ~105k (70% of 150k sample) | 74 | ~62 MB | ~130 MB |
| C (Combined) | ~205k | 196 | ~322 MB | ~650 MB |

**Decision**: Stratified subsample CIC-IDS2017 to 150,000 rows (preserving class proportions). This keeps memory feasible while retaining enough data for good model performance. Configurable via CLI flag.

## Implementation Plan

### New Files

| File | Purpose |
|------|---------|
| `crates/ids-preprocess/src/cicids.rs` | CIC-IDS2017 CSV loader: parse with header, trim columns, drop non-numeric, handle NaN/Inf, stratified sample, 70/12/18 split. Returns `DatasetSplit` |
| `crates/ids-preprocess/src/combined.rs` | `merge_datasets(nsl, cic) -> DatasetSplit`: zero-pad and concatenate |
| `crates/ids-engine/src/train.rs` | Training orchestrator: `train_pipeline(variant, data, config) -> TrainedPipeline`. Handles normalize → SMOTE → RF train → IForest train → ensemble → evaluate → save |
| `crates/ids-engine/src/bin/train_models.rs` | CLI binary with clap: `cargo run --bin train_models -- --models a,b,c` |
| `scripts/download_cicids.sh` | Download CIC-IDS2017 CSVs to `data/cicids2017/` |

| File | Change |

2. `cicids.rs` — CIC-IDS2017 loader + unit tests
3. `combined.rs` — dataset merger + unit tests
4. `lib.rs` (ids-preprocess) — module declarations and re-exports
5. `train.rs` — training orchestrator + unit tests
6. `lib.rs` (ids-engine) — module declaration
7. `Cargo.toml` (ids-engine) — clap + binary target
8. `train_models.rs` — CLI binary
9. `download_cicids.sh` — download script

### Training Pipeline (shared by all three models)

```
DatasetSplit
    → MinMaxScaler.fit_transform(x_train), transform(x_val, x_test)
    → SMOTE(x_train_scaled, y_train, target=5000, k=5)
    → RFModel::train(x_balanced, y_balanced)
    → IsolationForest::fit(x_train_scaled, trees=100, sample=256, contamination=0.05)
    → EnsembleDetector::new(rf_weight=0.7, iforest_weight=0.3)
    → evaluate(y_true, y_pred, n_classes=5)
    → save: rf.json, iforest.json, scaler.json, report.json
```

### Usage

```bash
# Download both datasets
cd cps-ids && ./scripts/download_dataset.sh       # NSL-KDD
cd cps-ids && ./scripts/download_cicids.sh         # CIC-IDS2017

# Train all three models
cd cps-ids && cargo run --release --bin train_models -- --models all

# Train individually
cd cps-ids && cargo run --release --bin train_models -- --models a
cd cps-ids && cargo run --release --bin train_models -- --models b
cd cps-ids && cargo run --release --bin train_models -- --models c

# Output directories
data/models/model_a_nslkdd/     # RF + IForest + scaler + report
data/models/model_b_cicids/
data/models/model_c_combined/
```

## Potential Challenges

| Challenge | Mitigation |
|-----------|------------|
| CIC-IDS2017 download URL instability (UNB mirrors go down periodically) | Script includes alternative source comments (Kaggle, IEEE DataPort) |
| Model C JSON serialization is large (~650 MB) | Keep subsample at 150k; document that users with <8 GB RAM should reduce to 100k |
| Label column name varies (" Label" vs "Label") | Case-insensitive, trimmed matching |
| Web Attack labels contain en-dash (U+2013) | Prefix match: `starts_with("Web Attack")` |
| SMOTE on combined dataset may be slow (205k rows × 196 features) | Use k=5, target=5000 per minority class; minority classes are small so SMOTE is fast |
