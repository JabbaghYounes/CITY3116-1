# Triple-Dataset Implementation Plan: NSL-KDD + CIC-IDS2017 + UNSW-NB15

This plan supersedes `dual-dataset-plan.md`. It adds the UNSW-NB15 dataset as a third data source and trains four model variants instead of three.

## Goal

Train four model variants using three IDS benchmark datasets spanning 1999â€“2017:

- **Model A**: NSL-KDD only (existing pipeline, unchanged)
- **Model B**: CIC-IDS2017 only
- **Model C**: UNSW-NB15 only (new)
- **Model D**: All three datasets combined

This allows the writeup to compare IDS performance across datasets from different eras, different feature extraction methodologies, and different attack landscapes â€” and to show that an ensemble trained on combined data can generalise across all three.

## Dataset Profiles

### 1. NSL-KDD (existing)

**Source**: University of New Brunswick (derived from KDD Cup 1999)
**Download**: https://www.unb.ca/cic/datasets/nsl.html
**Format**: CSV, no header, 43 fields (41 features + label + difficulty)
**Rows**: ~150,000 (train: ~125k, test: ~22k â€” official split provided)
**Features**: 41 (3 categorical: protocol_type, service, flag â†’ ~122 after one-hot)
**Labels**: 5 classes â€” Normal, DoS, Probe, R2L, U2R
**Year of traffic**: 1999

### 2. CIC-IDS2017 (from dual-dataset plan)

**Source**: University of New Brunswick, Canadian Institute for Cybersecurity
**Paper**: Sharafaldin, Lashkari & Ghorbani, ICISSP 2018
**Download**: https://www.unb.ca/cic/datasets/ids-2017.html (also Kaggle, IEEE DataPort)
**Format**: CSV with header, 8 day-split files generated by CICFlowMeter
**Rows**: ~2,830,000 (no official split â€” self-split required)
**Features**: ~78 (all numeric after dropping 4 non-numeric columns â†’ ~74 usable)
**Labels**: 15 (BENIGN + 14 attack types)
**Year of traffic**: 2017
**Data quality issues**: NaN/Infinity values, leading whitespace in column names, en-dash in Web Attack labels

See `dual-dataset-plan.md` for full CIC-IDS2017 details (feature list, quality issues, label mapping).

### 3. UNSW-NB15 (new)

**Source**: University of New South Wales, Canberra
**Paper**: Moustafa & Slay, "UNSW-NB15: a comprehensive data set for network intrusion detection systems", MilCIS 2015, IEEE
**Download**: https://research.unsw.edu.au/projects/unsw-nb15-dataset (also Kaggle, IEEE DataPort)

#### Format

- CSV files generated using Argus and Bro-IDS tools plus 12 custom algorithms
- Two distribution formats:
  - **Raw**: 4 CSV files (`UNSW-NB15_1.csv` through `UNSW-NB15_4.csv`) totalling ~2,540,000 records
  - **Official split** (recommended): `UNSW_NB15_training-set.csv` (175,341 rows) and `UNSW_NB15_testing-set.csv` (82,332 rows)
- Header row present
- Columns include feature values + `attack_cat` (category string) + `label` (binary 0/1)

#### Features (42 usable after dropping identifiers and targets)

**Dropped columns** (6): `srcip`, `dstip`, `sport`, `dsport`, `attack_cat`, `label`

**Categorical columns** (3 â€” require one-hot encoding):
- `proto`: Network protocol (~130+ unique values: tcp, udp, unas, ospf, etc.)
- `service`: Application service (~13 values: http, ftp, dns, ssh, smtp, etc.; `-` for none)
- `state`: Connection state (~13 values: FIN, INT, CON, REQ, RST, etc.)

**Numeric columns** (~39):

| Category | Features |
|----------|----------|
| Basic flow | `dur` (duration), `sbytes` (srcâ†’dst bytes), `dbytes` (dstâ†’src bytes), `sttl` (src TTL), `dttl` (dst TTL), `sloss` (src packets retransmitted/dropped), `dloss` (dst packets retransmitted/dropped) |
| Packet | `spkts` (srcâ†’dst packet count), `dpkts` (dstâ†’src packet count), `swin` (src TCP window), `dwin` (dst TCP window), `stcpb` (src TCP base seq num), `dtcpb` (dst TCP base seq num) |
| Content | `sload` (src bits/sec), `dload` (dst bits/sec), `smean` (mean src packet size), `dmean` (mean dst packet size), `sinpkt` (src inter-packet arrival time), `dinpkt` (dst inter-packet arrival time), `sjit` (src jitter), `djit` (dst jitter) |
| Temporal/connection | `tcprtt` (TCP round-trip time), `synack` (SYNâ†’SYN-ACK time), `ackdat` (SYN-ACKâ†’ACK time), `ct_srv_src` (connections to same service from src), `ct_srv_dst` (connections to same service to dst), `ct_dst_ltm` (connections to same dst in last 100), `ct_src_ltm` (connections from same src in last 100), `ct_src_dport_ltm`, `ct_dst_sport_ltm`, `ct_dst_src_ltm` |
| Binary/boolean | `is_ftp_login` (1 if FTP login), `is_sm_ips_ports` (1 if src=dst IP and port), `ct_ftp_cmd` (FTP command count) |
| Generated | `trans_depth` (HTTP request/response depth), `response_body_len` (HTTP response body size), `ct_flw_http_mthd` (HTTP methods count) |

After one-hot encoding proto (~130), service (~13), state (~13), the total feature count expands to approximately **~195 features** (39 numeric + ~156 one-hot). The exact number depends on the unique values present in the training split.

#### Attack Labels (9 categories + Normal)

| UNSW-NB15 Label | Mapped Class | Rationale |
|-----------------|-------------|-----------|
| Normal (label=0) | Normal (0) | Benign traffic |
| DoS | DoS (1) | Denial of service |
| Fuzzers | Probe (2) | Probing for vulnerabilities via random/malformed input |
| Analysis | Probe (2) | Port scanning, vulnerability scanning |
| Reconnaissance | Probe (2) | Network reconnaissance and information gathering |
| Generic | DoS (1) | Generic attack technique causing service disruption |
| Exploits | R2L (3) | Remote exploitation of vulnerabilities |
| Backdoors | U2R (4) | Maintaining persistent elevated access |
| Shellcode | U2R (4) | Code injection for privilege escalation |
| Worms | R2L (3) | Self-propagating remote malware |

#### Known Data Quality Issues

| Issue | Detail | Mitigation |
|-------|--------|------------|
| Duplicate records | Training set contains ~42% duplicated rows | Deduplicate on load |
| Missing values | Null values in `ct_flw_http_mthd`, `is_ftp_login`, `attack_cat` | Replace nulls with 0 for numeric, empty string for categorical |
| Class imbalance | Severe â€” Normal vs Worms imbalance ratio up to 514:1 | SMOTE oversampling for minority classes |
| Class overlap | Some records have ambiguous class boundaries | Accept as inherent dataset characteristic; document in writeup |
| High-cardinality categorical | `proto` has ~130+ unique values â†’ large one-hot expansion | Accept the expansion; RF handles high-dimensional sparse features well |
| `attack_cat` whitespace | Some values have leading/trailing spaces | Trim on load |

## Combined Model: Feature Space Design

### Approach: Union with Zero-Padding (extended to 3 datasets)

```
Combined vector = [NSL-KDD (122) | CIC-IDS2017 (74) | UNSW-NB15 (195)] = ~391 total

NSL-KDD sample:     [nsl_features | zeros(74) | zeros(195)]
CIC-IDS2017 sample: [zeros(122) | cic_features | zeros(195)]
UNSW-NB15 sample:   [zeros(122) | zeros(74) | unsw_features]
```

Feature names prefixed `nsl_`, `cic_`, `unsw_` to avoid collisions.

### Why This Still Works at ~391 Features

- **Random Forest**: Tree-based models are robust to high dimensionality with irrelevant features. Each tree only considers `sqrt(391) â‰ˆ 20` features per split, so zero-padded columns are rarely selected for samples where they carry no information. The RF effectively learns three separate rule sets and applies the right one based on which feature block is populated.
- **Isolation Forest**: Random feature selection at each split naturally adapts. The algorithm's strength is that anomalies are isolated quickly regardless of which feature subspace they occupy.
- **SMOTE**: Operates in the full 391-dimensional space, but synthetic samples are generated within each class, so the zero-padding structure is preserved (NSL-KDD synthetic samples still have zeros in CIC/UNSW blocks).

### Key Design Notes for `unsw.rs`

The UNSW-NB15 loader follows the same pattern as the NSL-KDD loader in `dataset.rs`:

1. Read CSV with header, trim all column names
2. Drop identifier columns (`srcip`, `dstip`, `sport`, `dsport`) and target columns (`attack_cat`, `label`)
3. Deduplicate records (drop ~42% duplicate rows in training set)
4. Handle nulls: replace missing numeric values with 0.0, missing categorical with empty string
5. One-hot encode 3 categorical columns (`proto`, `service`, `state`) â€” encoders fitted on training data only
6. Map `attack_cat` string to 5-class label via `unsw_attack_category()` function
7. Use official train/test split files directly; split training set 82/18 into train/val
8. Optional stratified subsampling via `max_samples` parameter
9. Return `DatasetSplit` with the shared `label_names()` (Normal, DoS, Probe, R2L, U2R)

### Key Design Notes for `combined.rs` (updated)

The merger now accepts a variadic number of datasets:

```
merge_datasets(splits: &[&DatasetSplit]) -> DatasetSplit

total_width = sum of all splits' feature widths

For each dataset i:
  pad_before = sum of widths of datasets 0..i
  pad_after  = sum of widths of datasets (i+1)..n
  x_padded   = [zeros(pad_before) | x_original | zeros(pad_after)]

Concatenate all padded splits, shuffle train/val, keep test ordered.
Feature names: prefixed with dataset identifier ("nsl_", "cic_", "unsw_").
```

## Potential Challenges

| Challenge | Mitigation |
|-----------|------------|
| Model D memory (~1.2 GB for feature matrix alone) | Subsample each dataset to 100k; configurable via `--max-samples` |
| Model D JSON serialization (~2.4 GB) | Acceptable for coursework; could switch to bincode for smaller files later |
| UNSW-NB15 `proto` column has ~130+ unique values | One-hot expansion is large (~130 columns) but RF handles it; alternatively limit to top-20 protocols and bucket the rest as "other" |
| UNSW-NB15 42% duplicate rows in training set | Deduplicate on load before any splitting or sampling |
| UNSW-NB15 download URL (UNSW CloudStor) may be unstable | Script includes Kaggle and IEEE DataPort as fallback sources |
| Combined feature space (~391 features) is large | `sqrt(391) â‰ˆ 20` features per RF split â€” the model naturally handles this; IForest also unaffected |
| Three different one-hot encoders (NSL-KDD, UNSW-NB15) need separate fitting | Each dataset loader fits its own encoders on its own training data; combined model receives already-encoded features |
| CIC-IDS2017 data quality (NaN/Inf, whitespace) | See dual-dataset-plan.md for full mitigations |
