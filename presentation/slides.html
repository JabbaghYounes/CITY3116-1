AI/ML Model Architecture

CNN+LSTM (~297K params)

  • Conv1d → ReLU → BatchNorm → Conv1d
  • LSTM (2 layers, 128 hidden)
  • FC (128 → 64 → 5 classes)
  • Dropout 0.3, early stopping
  • PyTorch → ONNX export for live inference

Ensemble (RF + Isolation Forest)

  • RF: 100 trees, max depth 20
  • IForest: unsupervised anomaly detection
  • Hard voting (0.7 RF / 0.3 IForest)

Evaluation & Recommendations

Strengths

  • CNN+LSTM 99.83% accuracy with 0.06% FPR
  • 4-layer defence-in-depth (rules + rate + CPS physics + ML)
  • Cross-dataset generalisation at 97.8% (Model D)
  • Real-time inference <1ms via ONNX Runtime
  • Ensemble improves minority class detection (R2L recall +7%)

Limitations

  • U2R class remains hardest (extremely rare samples)
  • Model A degraded by NSL-KDD binary test labels
  • Domain gap: training data ≠ live Modbus traffic
  • Rule engine generates false positives on localhost

Recommendations

  • Domain adaptation: fine-tune on captured Modbus traffic
  • Federated learning: train across CPS sites without sharing data
  • Adaptive thresholds: tune rule engine per deployment
  • SIEM integration: CEF/syslog export for SOC workflows

Standards Alignment

  • ISO 27001: A.12.4 logging, A.16 incident management
  • IEC 62443: defence-in-depth for industrial systems
  • NIST CSF: Detect & Respond functions
